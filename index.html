Lesson 1: The Basics of the Web and HTML

Basic Idea of the Web and How it Works

The main components of the web are computers, web browsers, servers, internet, content, and HTTP. The web is the entire collection of interconnected computers and servers which communicate with each other via a protocol known as HTTP. When an individual user visits a web page, the user’s web browser makes a HTTP request, which is sent over the internet, to the server hosting the appropriate HTML document. The server then sends back the HTML document to the user’s computer, which is then interpreted and displayed by the user’s web browser.

Computers are Stupid and Effective

Computers are able to perform tasks which are encoded via a computer language such as HTML. While able to perform complex tasks more effectively than humans, any little error in the code can have huge problems and prevent the code from performing the desired task. Computers read code literally with no room for interpretation or understanding.

HTML and HTML Elements

This first lesson detailed some of the key points of HTML, otherwise known as HyperText Markup Language. HTML documents comprise the majority of the content on the internet. HTML documents contain text content, which is the content seen by users, and markup, which controls how the text content looks. Each HTML document is comprised of HTML elements. These elements come in different sorts and are determined via tags.

Types of Tags

“B” tag - Makes the text content bold.
“M” tag - Makes the text content initialized.
“Break” tag - Makes an one line break, has no closing tag.
“Paragraph” tag - Creates an "invisible box" around the text content.
“Anchor” tag - Makes a link to another page.
“Img” tag - Tags an image.
"Div" tag" - Makes a box around content which allows for markup of the content.
"Span" tag - Allows for markup of a single line of element.

Element and Tag Lingo

Inline/Block- Elements come in either sort. Block elements make an "invisible box" around text content and can have a determinable height and length.
Void Tag - A tag with no closing tag.

Lesson 2: Creating Structure in a HTML Document

Web Development Languages

There are three languages used for web page development, HTML, Java, and CSS. HTML gives structure and content to a page. CSS, otherwise known as Cascading Style Sheets, gives style to the page elements, such as font size, text color, background, position, border, and etc. Java gives the web page an interactive element. While not obvious at first, every web page on the internet is comprised of boxes. Boxes are defined with the "Div" tag. To add style to each box, give each element a class attribute.

The Structure of Web Pages

To see the structure of a web page, in Google Chrome use "Developer tools." What is shown is the structure of the web page as interpreted by the web browser. Generally speaking, the structure shown is described as a tree structure. This is because HTML elements can have other elements nested inside them, and so on and so forth. When examining a web page's code, all the code within the page can be modeled in a Document Object Model (DOM), which is a tree-like structure of a page's code. A good summary about tree structures can be found here. More information about DOMs can be found here.

Translating a Web Page Design into Code

Since every web page is comprised of boxes, start by Boxify a web page design, going from big boxes to small boxes. Boxes can exist within other boxes. The same web page design can be boxifed in different ways.

Lesson 3: Adding CSS Style to HTML Structure

Importance of Avoiding Repetition

It's a good habit to avoid repetition of code whenever possible. This makes your code easier to read and reduces the chances of errors. As an example, when using lots of repetition of styling code, it not only means a greater chance of error, but makes it more difficult to make markup changes to all desired elements. To make a simple markup change, instead of changing a small amount of code in the CSS file, you would have to change the CSS code in each one of the desired elements. This in turn leads to a greater chance of error if some of the elements are missed or incorrectly modified. Why make your coding experience harder?!?

CSS Styling Lingo

Selector - Term used to select all elements of a particular sort in a HTML document.
Class Selectors - Term used to select only a certain elements of a particular sort in a HTML document.

Adding CSS styling to a HTML Document

There are several ways to add CSS styling code to a structured HTML document:

1) Write the CSS styling code in a "Style" tag nested in the “Head” tag of the HTML document. This is fine for very small projects.

2) Link the HTML document to a separate CSS file. Both files are placed in the same project folder. Helps keep organization for larger projects. A “Head” tag is placed at the top of the HTML document with a "Link" tag inside.

3) Adding CSS styling in line with the HTML code. Very bad idea, leads to lots of repetition of styling code, more chances for errors in code.

Stage 2 Notes

Lesson 1: Introduction to Serious Programming

What are Computers?

An universal machine, that can be programmed to do almost anything. Whatever we can write a program for can be done by a computer. Originally, computers were thought to only be useful for mathematical calculations, only later were they used for other applications.

What are Programs?

A precise sequence of steps or instructions that tell the computer what to do. Programs are written by programmers via a programming language such as Python.

More about Python

Python is one of many different programming languages, each with different applications, and with its own particular grammar, and commands.

Programming Languages vs. Natural Languages

Programming languages were invented because of shortcomings of natural languages, ambiguity, and verbosity. Natural languages are often ambiguous were the same sentence or word can have different meanings. We want the computer to interpret the code written to be exactly as intended by the programmer. Natural languages are also very verbose, aka wordy. To write a program for a complex task in a natural language would require a lot of text. While the same task in a computer language may only take a few lines of code.

Grammar in Programming Languages

Like natural languages, programming languages have a grammar that allow programmers to form “valid” sentences and defines what strings are possible in the given programming language. Python's grammar is much stricter than natural language grammar. Python requires code be written with a very specific syntax or grammar. Code must match the Python grammar exactly, otherwise "bugs" or errors would arise.

Commands and Functions in Python

Print - A command that prints outs the result of a string of code.
# - These are comments in Python, and ignored by the Python interpreter.
Find - Function gives the position, via a number, of the target string in the search string. If the target string is not found in the search string, the output will be -1.
Indexing - Function that extract parts from a target string.

Lesson 2: Variables and Strings

Variables in Python

In Python, variables refer to the process were Programmers assign a name to a value. This is done through a variable assignment. The use of variables makes code easier to read, and allows for easier changes in values. A variable can also refer to an assignment of a name to a string, and expression. As a convention, variable names should be in lowercase.

The meaning of the "=" sign in Python

The equal sign, "=" has a very different meaning in Python and many other programming languages. Unlike in Mathematics, where the equl sign means "equality," in Python the equal sign means "to be assigned to."

Python Lingo

String - A sequence of characters.
Subsequence - String taken from an original string

Lesson 3: Input --> Function --> Output

Functions in Python

Conceptually, functions work in programming in a manner similar in Mathematics. A function takes in an input, performs an operation, and delivers an output. There can be any number of inputs in a function, but the number of inputs has to agree with the expected number in the procedure. Inputs are also called operands, and arguments.

Making a Function in Python

Functions in Python are started with the word def. The function is then given a name followed by parameters in parentheses. The parameters are the inputs for the function and are later replaced by actual values when the function is used. The operation of the function is signaled by the keyword return which tells Python what to produce as an output. If a return statement is omitted from the function, the function will produce a "None" as an output.

Lesson 4: Control Flow & Loops

Making Decisions and Complex Programs

To make programs of greater complexity and usefulness, we need to incorporate the use of arithmetic, if statements, loops, comparisons, and procedures. With these we're able to write code that will behave differently based on decisions. We can also use these to simulate any other computer program.

Making Comparisons in Python

To make comparisons in Python, the operators are used. When making comparisons between values, the output given by Python is a Boolean value, which is either the value "True" or "False."

If statements

By using comparisons, we can make decisions in Python through the use of if statements. To start an if statement, we start with the keyword “if” followed by a test expression. The indented block is the code the function will execute if the comparison is true. If the test expression is not true, then the block will not execute. The end of the if statement is signaled by indentation. The next statement that is not indented will run regardless whether or not the test expression is true.

Max and Or Operators

The or operator can sometimes be used in place of an if statement. The output given by the or operator will be a Boolean value. If the first expression evaluates to true, the value of the or expression is true and the second expression is not evaluated. If the first expression evaluates to false, the value of the or expression is taken from the value of the second expression.

The Max operator returns the largest number from a set.

Loops

Loops allows the coder to do things over and over again. While loops have both a test expression and indented block. If the test expression is true, the function will execute the block. This process is "looped" over and over again so long the test expression evaluates to true.

Lesson 5: Debugging

Bugs Happen and that's OK!!

Bug are an inevitable part of programming for programmers of all skill levels. Instead of trying to avoid bugs entirely, a programmer should develop a strategy, or mental model to systematically get rid of them.

Debugging is more than just correcting errors and learning how you made a mistake. It's a scientific process in examining code to discover things you didn't expect in it. Being able to find errors is a big part to writing software.

Guide to Debugging

Traceback Messages- When a program crashes. Usually an error message called a traceback is given by Python. The traceback tells you what line the code crashed on and what file it was running.

Example Code- If working example code is available use the example code as a base and step by step make the changes until you arrive to the result you wanted. Then compare your new code with your original code to see where the original code had the bug.

Keep Old Version- Keeping old versions of your code allows you to compare and share your code with others who can help you find the bug.

Print Statements- Adding print statements to your program can show where in the code the program begins to misbehave. Thus allowing you to pin point the location of the bug.

Lesson 6: Lists & For Loops

Lists

Lists are a type of structured data that let us organize information. The list is identified by square brackets, and the elements are separated by commas. The elements could be strings, numbers, characters, other lists.

Mutations

Lists support mutation, which allows the coder to change the value of list after it has been created. Whenever the value of an element is changed, the list is being modified, nor is a new list created.

Lists Lingo

Aliasing-When two variables refer to the same object.

List Operations

Append-Adds a new element to the end of the list by mutating the old list.

Plus-Produces a new list by the adding the elements from two old lists. Does not mutate old lists.

Len-Returns the length, or number of elements in a list. Also works on strings or any object that is a collection of things.

Index-Returns the first position where a value is located in a list. If not found, produces an error.

In-If a value is in the inquired list, returns true. Otherwise returns false.

Not In-If a value is not in the inquired list, returns true. Otherwise returns false.

For Loops

For Loops go through each element of a list and executes a block of code on each element, then loops back to the next element in the list. After the last element in the list, the for loop is complete and Python goes to the next line of code.

Lesson 7: How to Solve Problems

What do Problems Have in Common?

All computational problems have in common inputs and outputs. Problem are defined by possible inputs, usually an infinite set, and a relationship between the inputs ⇒ and desired outputs. A solution will be a procedure that can take any input from the set and produce a desired output that satisfies the relationship we want.

Rules to Help Solve Problems

What are the inputs? What is the set of valid inputs? How are the inputs represented?

What are the outputs? How should we specify the outputs? Note if we return an output, we can do further computations with the output. If we print the output it’s hard to do anything else with the output.

Work through some examples by hand. Maybe start with “pseudo code” on paper. Figure out how to map the inputs to the correct outputs. Try to figure out the relationship between the two.

Simple mechanical solution. Consider systematically how a human can solve the problem on paper. Don’t optimize prematurely, try a simpler method first. Try to breakdown the problem.

Develop incrementally and test as you go. Write small bits of code that can be tested as you go.
